# Revision del bot - 2026-06-15

## Hallazgo critico corregible

En `src/ai.js`, la funcion local `parseDate` evalua `manana` antes de `pasado manana`.

Eso puede provocar que un mensaje como:

```text
quiero cita pasado manana
```

se interprete como manana en vez de dos dias despues.

## Parche recomendado

Cambiar este bloque:

```js
if (/\bhoy\b/.test(text)) return formatISODate(today);
if (/\bmanana\b|\bmañana\b/.test(text)) return formatISODate(addDays(today, 1));
if (/\bpasado manana\b|\bpasado mañana\b/.test(text)) return formatISODate(addDays(today, 2));
```

por este:

```js
if (/\bhoy\b/.test(text)) return formatISODate(today);
if (/\bpasado manana\b|\bpasado mañana\b/.test(text)) return formatISODate(addDays(today, 2));
if (/\bmanana\b|\bmañana\b/.test(text)) return formatISODate(addDays(today, 1));
```

## Por que importa

Si una paciente pide cita pasado manana, el bot puede ofrecer horarios del dia equivocado. En un bot de agenda esto es grave porque genera confusiones y puede terminar en citas falsas o mal fechadas.

## Otro punto a vigilar

Cuando el bot responde `ese horario se acaba de apartar`, normalmente viene de `appointment_locks` en Supabase. Si no aparece nada en Google Calendar, hay que revisar locks activos en `/debug/config` o en el diagnostico del inbox.

Recomendacion operativa: antes de probar otra vez, limpiar locks expirados o esperar los minutos de `APPOINTMENT_LOCK_MINUTES` si hubo una prueba fallida.
